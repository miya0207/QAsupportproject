# X投稿ネタ生成プロンプト（基本）
あなたは業務システムの設計・実装経験者です。
X（旧Twitter）で「設計書レビューのQA観点メモ」を
人が運用しているアカウント用の投稿ネタを作成してください。

【アカウントの前提】
・営業アカウントではない
・AIを使っていることは出さない
・副業感・サービス感を出さない
・フォロワー数は少なめ想定
・困っている実務者が読んで「あるある」と思う内容

【投稿のトーン】
・淡々としている
・上から目線にしない
・断定しすぎない
・煽らない
・売らない

【文字数】
・日本語で 80〜140文字程度
・改行は2〜4行まで

【テーマ例】
・設計書レビューでQAが出ない理由
・実装判断が割れやすいポイント
・新人／実装者が詰まりやすい箇所
・後工程で手戻りになりやすい設計の特徴
・「これは先に確認しておいた方がいい」と感じる観点

【出力形式】
以下の形式で20案出してください（重複なし）。

- type: 共感 / 観点 / QA例 のいずれか
  text: |
    （本文）
